Artifact 2 Narrative

The original artifact was several classes (tasks, appointments, contacts) in a java project that has parameters, like name, description, etc and the purpose of the original project was to use and test JUnit. This project was created for CS-320 and was created in June 2025. There was no output or display and it was solely used for testing purposes only. The enhancements I did were to expand the project significantly. I created an API using SpringBoot and created an HTML page that allows the user to see the data from the original java. I also changed the hashmap storage to more permanent storage by using MongoDB to create three different databases for each of the original classes.

I included this artifact because it uses Java, a language that is extremely common in real world scenarios, as well as because the original artifact used hashmaps to store the data which created a foundation for me to expand upon with different algorithms and data structures. For each of the databases on their own individual pages I added a sort and search algorithm. I used bubble sort and linear search, as these are not the fastest in the given scenario I felt as though they give the best display to the user of how they work as clicking the buttons allows the user to see the bubble sort in action as well as searching in the bar allows the user to see the algorithm check each database entry.

The outcomes I had planned were to display techniques and skills in practice to deliver industry standard goals as well as design and implement solutions using algorithmic principals to meet goals while weighing the benefits and the downsides. I feel as though I have met these goals especially when concerning algorithms as I had to weigh the benefits and downsides of including faster more efficient sorting algorithms or choosing to go with algorithms that are easier to demonstrate to the user. I also used tools like SpringBoot to my advantage as an easy way to let the user see the databases and how they work in conjunction with the algorithms.

When creating this artifact a big issue I ran into was deciding which algorithms I should implement. I knew that bubble sort had a time complexity of O(n^2) while other sorting algorithms like merge sort have a time complexity of O(n log n) which is much faster. In the end however I felt it best to be able to fully demonstrate the data sorting without having to look under the hood to see exactly how it is done. I also included linear search because the length of the data is relatively small and is also unsorted so I felt this would be the best move. I was able to learn in doing this different types of sorting and searching in action as well as how to use SpringBoot and how it functions as I only have experience with Express in the past.

Artifact 3 Narrative

The original artifact was a database that used python functions in order to perform CRUD operation (CREATE, READ, UPDATE, DELETE) on a database of animals. The functions were made using pure python code and the database was displayed using a Dash server. The enhancements changed the database and uses HTTP operations as opposed to purely Python ones. The database is now displayed on a local server and allows users to perform all the CRUD actions. A big part of the enhancements was adding a login function which allows users to create an account using a username and password and the credentials are stored in a database with the password being hashed. The database cannot be viewed by public users only those who have signed in. The web application uses a flask backend with a basic use of HTML, CSS, and JavaScript as well as MongoDB to store the animal information as well as the credentials of the users.

I chose this artifact for my portfolio because of the use of Python as the backend as well as the implementation of JWT tokenization in order to authenticate users. Flask while being a lightweight backend is a good way to demonstrate how useful Python can be used. Authentication is super important and having a way to register and login to a site as well as being able to keep passwords safe so outside sources can not get them was also a goal to be demonstrated.

The outcomes I had planned to align with were to use skills, techniques, and tools in order to meet the goals at an industry standard level, building environments that allow and promote collaboration, and have a security mindset that predicts and prevents any vulnerabilities and protects data and resources. I have met all of these outcomes in this artifact. I used HTTP and python to create a functioning web app that uses CRUD operations which are at the industry standard. I also created a login that hashes passwords and uses JWT tokens to keep the user signed in for a set amount of time without having to store permanent data on the web page.

I initially had a challenge of making sure that my tokens worked and that they stayed in the local storage on the webpage. I learned how to make them work however as well as I solidified my understanding of hashing passwords as well as using Flask as a backend. I was also able to better understand the whole structure of the project. I knew how routing and api calls worked before but now I was able to implement them with little to no mistakes.
